서비스 대시보드

서빙의 서비스 정보를 확인 할 수 있고 상태를 변경시킬 수 있는 대시보드 화면입니다.

서비스 대시보드 화면#

1. 모델 상세 공통#

  • 모델 상세 공통 부분입니다.
    • 모델 상세 화면으로 이동합니다.

2. 기본 정보 설정#

  • 모델 엔드포인트 정보를 확인하고 내용을 복사할 수 있습니다.

3. 서빙 설정#

  • Default/Canary 의 서빙 정보를 보여주고 상태를 변경할 수 있게 합니다.

  • Default의 상태 변경은 중지/일시정지/재시작 입니다.

  • Canary의 상태 변경은 배포중지/프로모션 입니다.

  • Default/Canary의 상태 변경을 하면 서비스 대시보드나 버전현황에서 확인가능 합니다.

4. 로그 조회#

  • 디폴트 서빙과 Canary에 대한 모델 로그를 조회합니다 버전 별 로그를 선택할 수 있습니다.

5. 서빙 설정#

  • Default만 서빙 중 일 때
    • Default 서빙의 설정을 업데이트 할 수 있습니다.
  1. 서빙 버전
    • 서빙 할 모델 버전입니다.
  2. Concurrency
    • 서빙 할 모델의 Concurrency 값 입니다.
  3. 자원설정 - 배포자원
    • 각 자원 당 요청한 자원 크기/배포자원에서 사용가능 크기를 나타냅니다.
  4. 자원설정 - 자원타입
    • Small, Medium, Large, 직접설정이 있으며 각 자원당 요청자원 크기가 달라집니다.
    • 직접설정에서 limit 값이 요청자원의 크기에 영향을 줍니다.
  5. Replica
    • Replica 값의 최소 ~ 최대를 설정한다. 요청자원의 크기에 최대값이 영향을 줍니다.
  6. GPU
    • GPU 값을 설정합니다.

  • Canary 배포 중 일 때

    • Canary가 배포 중 일 때는 Canary 설정 만 변경 가능 합니다.

  1. Canary Traffic Percent
    • Canary Traffic Percent 설정 부분으로 Default 와 100% 중 나누어 가지게 됩니다.
  2. Default 서빙 자원
    • Default 서빙 자원 부분으로 설정이 불가능 합니다.
  3. Canary 서빙 자원 설정
    • Default 서빙 업데이트와 동일한 내용입니다.

6. 대시보드#

  • 대시보드 형태로 각 자원의 상태를 표시합니다.
