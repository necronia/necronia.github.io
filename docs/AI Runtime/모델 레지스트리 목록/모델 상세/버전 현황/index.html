버전 현황

모델의 버전을 추가하고 서빙 사작 및 모델 버전 별 상태를 확인할 수 있는 화면입니다.

버전 현황 화면#

1. 모델 상세 공통#

  • 모델 상세 공통 부분입니다.
    • 모델 상세 화면으로 이동합니다.

2. 모델 버전#

  • 모델의 버전들을 표시하고 모델 버전 등록으로 버전을 추가할 수 있습니다.

  • 모델 등록 버전에서 항목을 설정하고 등록하면 버전이 추가 됩니다.

  • 설정방법은 모델 등록 화면과 유사합니다.

3. 모델 상태#

  • 모델 등록 후 모델상태는 빌드 중/배포가능/빌드실패가 있습니다.
  • 모델 상태가 배포가능이면 서빙시작을 할 수 있습니다.

4. 서빙/서빙 상태#

  • 서빙 동작을 실행시킬 수 있고 서빙 상태를 표시합니다.

4-1. 서빙 시작#

  • 서빙 중인 모델이 없을 경우 서빙 시작으로 디폴트 서빙을 시작할 수 있습니다.

  1. 서빙 버전

    • 서빙 할 모델 버전입니다.

  2. Concurrency

    • 서빙 할 모델의 Concurrency 값 입니다.

  3. 자원설정 - 배포자원

    • 각 자원 당 요청한 자원 크기/배포자원에서 사용가능 크기를 나타냅니다.

  4. 자원설정 - 자원타입

    • Small, Medium, Large, 직접설정이 있으며 각 자원당 요청자원 크기가 달라집니다.
    • 직접설정에서 limit 값이 요청자원의 크기에 영향을 줍니다.

  5. Replica

    • Replica 값의 최소 ~ 최대를 설정한다. 요청자원의 크기에 최대값이 영향을 줍니다.

  6. GPU

    • GPU 값을 설정합니다.

3-2. 서빙 상태#

  • 서빙 시작 후 서빙 상태가 표시합니다.

  • 서빙 상태가 RUNNING 이면 Canary 배포가 가능합니다.

3-3. 에러정보 팝업#

  • 서빙 상태가 ERROR로 표시될 때 ERROR를 선택하면 에러정보 팝업이 띄워서 에러 내용을 보여줍니다.

4. Canary 배포/Canary 배포 상태#

  • Default 서빙이 RUNNING 상태일 때 Canary 배포 추가를 할 수 있습니다.

4-1. Canary Traffic Percent#

  • Canary Traffic Percent 설정 부분으로 Default 와 100% 중에서 나누어 가지게 됩니다.

4-2. Default 서빙 자원#

  • Default 서빙 자원 부분으로 설정이 불가능 합니다.

4-3. Canary 서빙 자원 설정#

  • 서빙 시작과 동일하기 때문에 4-1을 참조하시면 됩니다.

5. 변경일시#

  • 해당 버전의 상태가 변경된 일시 입니다.

6. 컨텍스트 메뉴#

  • 모델 등록 정보
    • 모델 등록 빌드 정보에 대해 표시합니다.
  • 워크 플로우 조회
    • 모델의 워크 플로우를 조회합니다.
  • 모델 버전 삭제
    • 모델 버전을 삭제합니다.

7. 모델 버전 상세 정보#

  • 버전을 선택하면 해당 버전의 모델의 상세 정보를 표시합니다.
