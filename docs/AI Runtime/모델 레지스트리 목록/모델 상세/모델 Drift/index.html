모델 Drift

모델 Drift는 모니터링을 등록하고 Drift 결과 및 정보를 보여준다.

모델 Drift 화면#

1. 모델 상세 공통#

  • 모델 상세 공통 부분입니다.
    • 모델 상세 화면으로 이동합니다.

2. 새로 고침#

  • 새로 고침을 수행합니다.

3. 모니터링 등록#

  • 모니터링을 등록할수 있습니다.

  1. 모니터링 설명

    • 모니터링에 대한 설명을 작성합니다.

  2. 기준시각

    • 모니터링을 수행할 기준시각을 설정합니다.

  3. 알림설정

    • 등록된 Slack 채널을 이용하여 알림설정을 on/off 할 수 있습니다.

  4. 분석주기

    • 분석주기는 기본방식과 Cron 표현식으로 설정할 수 있습니다.
      • 기본방식
        • 기준시각을 기준으로 일/주/워로 분석주기를 설정할 수 있습니다.
      • Cron 표현식
        • Cron 표현식을 이용하여 분석주기를 설정할 수 있습니다.

  5. 기준 데이터

    • 모델에 학습 데이터가 있을 때에는 학습 데이터와 과거 데이터 중 기준 데이터를 설정할 수 있습니다.

  6. 대상 피쳐 설정

    • 피쳐 설정을 통하여 피쳐의 값을 설정할 수 있습니다.
      • On/Off 스위치를 통하여 해당 피쳐를 Skip 할 수 있습니다.
      • 임계값 일괄설정 체크하시면 일괄적으로 값을 설정할 수 있습니다.
      • 피쳐의 타입설정을 할 수 있습니다.
      • Add 버튼으로 사용자가 새 피쳐를 추가할 수 있습니다.

4. 모델 Drift 항목#

  • 설명/분석주기/드리프트 여부/모델상태/기준 데이터/최근 실행 일시/다음 실행 일시 를 알려 줍니다.

5. 모델 Drift 상세#

  • 모델 Drift 항목을 선택을 하면 모델 Drift 상세를 확안할 수 있는데 드리프트 결과/정보로 나누어져 있고 탭을 통해서 선택이 가능합니다.
  • 드리프트 결과 탭
  1. 모니터링 정보 표시
    • 새로고침버튼으로 새로고침을 할 수 있습니다.
    • 모니터링 ID/모니터링 상태/최근실행 일시/다음 실행 일시/드리프트 여부를 표시합니다.
  2. 상위 드리프트 피쳐
    • 상위 Top3의 드리프트 피쳐를 보여 줍니다.
  3. 드리프트 추이
    • 선택한 날짜와 피쳐에 관한 드리프트 추이를 그래프로 확인할 수 있습니다.

  4. 드리프트 이력
    • 드리프트 실행이력을 확인 할 수 있고 실패이력 표시로 실패이력만 확인 가능합니다.
  • 드리프트 정보 탭
  1. 드리프트 정보 표시
    • 새로고침버튼으로 새로고침을 할 수 있습니다.
    • 드리프트 설명/가준시간/알림설정을 표시합니다.
  2. 분석주기
    • 설정된 분석주기를 표시합니다.
      • 기본 방식으로 표시될 때
      • Cron 표현식으로 표시될 때
  3. 기준 데이터
    • 기준 데이트의 상태를 표시해 줍니다.
  4. 대싱 피쳐
    • 대상 피쳐들의 상태를 표시해 줍니다.
  5. 수정
    • 등록과 다르게 알림설정/분석주기/대상 피쳐 만 수정 가능합니다.
      • 알림설정/분석주기 수정
      • 대상 피쳐 수정
  6. 새로고침
