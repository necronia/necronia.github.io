대시보드

워크플로우 작업 현황인 스케줄, 모델 생성 상태, Sandbox 동작 수 등을 확인 할 수 있고 작성된 프로젝트, 워크플로우, 모델, 리포트를 확인 할 수 있습니다.
서비스 관리자의 경우 자원관리를 통해 현재 사용중인 자원 상태까지 확인 할 수 있습니다.

1. GNB 메뉴#

  1. 프로젝트, 워크플로우, 애셋에서 검색하여 결과 표시
  2. 프로젝트 리스트로 이동
  3. 글로벌 애셋 리스트로 이동
  4. 소스 저장소로 이동
  5. AI Runtime 서비스로 이동
  6. 가이드가 있는 화면일 경우 가이드 센터 표시

    Accuinsight+ 3.0에 대하여 - Document 사이트로 이동
    현재 화면 둘러 보기 - 가이드 포인트와 설명이 화면에 표시
  7. 사용자 메뉴 표시

2. 사용자 메뉴#

  1. 사용자 상세 페이지로 이동
  2. 서비스 관리자에게만 노출되는 메뉴이며 관리자 콘솔 페이지로 이동
  3. 로그아웃 수행

3. 보기 모드#

카드 보기 모드와 리스트 보기 모드에 대해서 토클 수행
왼쪽 - 카드 보기 모드, 오른쪽 - 리스트 보기 모드

4. 워크플로우 현황#


오늘 기준으로 워크플로우 현황을 표시

  1. 필터 - 전체 워크플로우, 내 프로젝트 워크프로우, 내가 작성한 워크플로우
    • 전체 워크플로우 - 전체 워크플로우에서 현황을 수집
    • 내 프로젝트 워크플로우 - Owner 권한이 있는 프로젝트의 워크플로우에서 현황을 수집
    • 내가 작성한 워크플로우 - 내가 작성한 워크플로우에서 현황을 수집
  2. 워크플로우의 스케줄 현황 표시
  3. 생성중인 AutoML, Labelling, Clustering 현황 표시
  4. 상태가 Running인 Sandbox (Jupyter, R) 현황 표시

5. 자원관리#


Accuinsight+ 3.0에 등록되어 있는 클러스터 자원 현황 표시

info

서비스 관리자일 경우에만 표시가 됩니다.

  1. Chevron - 자원현황 표시를 열거나 닫음
  2. 클러스터 필터 - 등록된 클러스터별로 필터 수행
  3. 클러스터 자원 상태 표시
    • 경고 - CPU, Memory, GPU 사용이 경고(90% 이상)인 항목 수
    • 주의 - CPU, Memory, GPU 사용이 주의(70% 이상)인 항목 수
    • 정상 - CPU, Memory, GPU 사용이 정상(70% 미만)인 항목 수
  4. 클러스터의 CPU, Memory, GPU 사용 현황 표시
  5. 네임스페이스 필터 - 클러스터에 생성된 네임스페이스 필터 수행
  6. 네임스페이스 자원 상태 표시
    • 경고 - CPU, Memory, GPU 사용이 경고(90% 이상)인 항목 수
    • 주의 - CPU, Memory, GPU 사용이 주의(70% 이상)인 항목 수
    • 정상 - CPU, Memory, GPU 사용이 정상(70% 미만)인 항목 수

6. 프로젝트#

  1. View 권한 이상 포함된 모든 프로젝트 수를 표시
info

서비스 관리자의 경우 프로젝트 권한과 상관 없이 모든 프로젝트의 수가 표시 됩니다.

  1. 프로젝트 리스트로 이동
  2. 프로젝트 생성 페이지로 이동
  3. 북마크 설정/해제
info

북마크 설정 시 우선적으로 정렬되어 리스트 상위에 표시 됩니다.

  1. 프로젝트의 포함된 워크플로우, 모델, 리포트 수를 표시
  2. 프로젝트의 More Actions 메뉴
    • 새 워크플로우 시작 - 워크플로우 생성 화면으로 이동
    • 기본정보 편집 - 프로젝트 수정 화면으로 이동
    • 프로젝트 복제 - 프로젝트 복제 수행
    • 삭제 - 프로젝트 삭제 수행
info

프로젝트 구성요소(워크플로우, 모델)를 먼저 삭제해야 삭제가 가능 합니다.

7. 워크플로우#

  1. View 권한 이상 포함된 모든 프로젝트의 소속된 워크플로우 수를 표시
info

서비스 관리자의 경우 프로젝트 권한과 상관 없이 모든 워크플로우 수가 표시 됩니다.

  1. 워크플로우 리스트로 이동
  2. 북마크 설정/해제
info

북마크 설정 시 우선적으로 정렬되어 리스트 상위에 표시 됩니다.

  1. 워크플로우에 스케줄러, 리포트가 포함되어 있으면 아이콘 표시
  2. 워크플로우의 More Actions 메뉴
    • 편집 - 워크플로우 상세 화면으로 이동
    • 복제 - 워크플로우 복제 수행
    • 삭제 - 워크플로우 삭제 수행
info

편집 시 Editor 권한이 없을 경우 읽기 전용 모드로 진입 합니다.

8. 모델#

  1. View 권한 이상 포함된 모든 프로젝트의 소속된 모델 수를 표시
info

서비스 관리자의 경우 프로젝트 권한과 상관 없이 모든 모델 수가 표시 됩니다.

  1. 모델 리스트로 이동
  2. 북마크 설정/해제
info

북마크 설정 시 우선적으로 정렬되어 리스트 상위에 표시 됩니다.

  1. 모델 이름으로 AI Runtime 배포 표시 (회색일 경우 미 배포, 파란색일 경우 배포 됨)
  2. 모델의 More Actions 메뉴
    • AI Runtime 이동 - AI Runtime 서비스의 모델 배포 현황 페이지로 이동
    • 편집 - 모델이 작성되어 있는 워크플로우 상세 화면으로 이동
info

AI Runtime 이동 시 배포가 삭제되었을 경우 오류 메시지가 발생할 수 있습니다.
편집 시 Editor 권한이 없을 경우 읽기 전용 모드로 진입 합니다.

9. 리포트#

  1. View 권한 이상 포함된 모든 프로젝트의 소속된 리포트 수를 표시
info

서비스 관리자의 경우 프로젝트 권한과 상관 없이 모든 리포트 수가 표시 됩니다.

  1. 리포트의 More Actions 메뉴
    • 편집 - 리포트 편집 화면으로 이동
info

편집 시 Editor 권한이 없을 경우 읽기 전용 모드로 진입 합니다.
