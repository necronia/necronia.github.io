Pattern Parser

Grok 패턴을 활용한 Parser.

워크플로우 보드에서 데이터 노드를 선택하고 노드 메뉴에서 [전처리] 카테고리의 [기본처리]노드중 [Pattern Parser]노드를 선택후 우측 패널에서 항목을 입력합니다.

우측 패널의 [더 보기] 버튼을 누르면 입력가능한 전체 항목을 볼 수 있습니다.

워크플로우 보드에서 노드를 우클릭하면 메뉴항목을 볼 수 있습니다.

우측 패널#

  1. targetColumn : 목록 중 적용할 컬럼 선택
  2. schema : 신규 컬럼명. 패턴 찾기를 통해 작성 또는 직접입력 (적용 패턴이 여러 개일 경우 우측 [+]버튼으로 추가)
  3. pattern : Grok 패턴
  4. parsedSample : pattern으로 치환되지 않은 문자열, 패턴 찾기 다이얼로그에서 pattern으로 치환가능
  5. 패턴 찾기 팝업 호출
  6. 추가 : 입력된 값을 적용하여 데이터 노드를 생성

메뉴#

  • 이름바꾸기 : 작성 중인 노드명 변경
  • 즐겨찾기에 추가 : 즐겨찾기에 [Add Condition Column]노드 추가
  • 삭제 : 해당 노드를 워크플로우에서 제거

패턴 찾기#

targetColumn의 첫번째 데이터로 패턴 찾기 팝업에서 진행

  1. 패턴 입력 : targetColumn의 첫번째 데이터 로드
  2. 검색 : 패턴입력에 있는 문자열에서 패턴을 추출
  3. 패턴 입력과 패턴 최종 결과를 targetColumn의 첫번째 데이터로 초기화
  4. 패턴 중간결과 : 검색된 패턴 리스트
  5. 적용 패턴 선택 : 패턴 입력에서 검색된 패턴중 적용할 문자열, 형식을 선택
  6. column name : 적용 컬럼명 입력
  7. 다음 : 선택된 적용 패턴의 문자열을 입력된 column name으로 패턴 최종결과를 replace, 적용된 패턴의 문자열은 패턴 입력에서 자동 삭제
  8. 패턴 최종결과 : 우측 패널 [pattern]에 적용할 최종 문구
  9. 취소 : 입력된 내용을 적용 하지 않고 팝업 닫기
  10. 적용 : 입력된 내용을 적용하여 우측 패널의 [schema], [pattern], [parsedSample] 항목 자동 입력

예제#

쇼핑몰 로그 데이터를 Parsing하는 워크플로우 작성

예제 데이터 위치 : https://github.com/AccuInsight/accuinsight_handson/blob/master/Accuinsight%2B_v2.0/Pipeline/handson/2.%20%EC%87%BC%ED%95%91%EB%AA%B0%20%EB%A1%9C%EA%B7%B8%20%EB%B6%84%EC%84%9D%20%EB%B0%8F%20%EC%83%81%ED%92%88%20%EC%B6%94%EC%B2%9C/Files/shopping_web.log

  1. 워크플로우에 [쇼핑몰 로그 데이터]를 로드

    • [쇼핑몰 로그 데이터]노드의 우측 패널에서 부가옵션 설정 클릭
    • format : csv
    • delimeter : ,
    • option : key -> header , value -> false

  2. [쇼핑몰 로그 데이터] 노드에 [Pattern Parser] 노드를 연결하여 데이터셋을 생성

    • targetColumn을 '_c0' 선택
    • 패턴찾기 팝업 호출 하여 schema, pattern 입력
      • [1], [2], [3], [4] 순으로 다음과 같이 적용
        • HOST - matches text : 141.146.8.66 - column name : host
        • WORD - matches text : yqokwuaz6a - column name : user_id
        • TIMESTAMP_ISO8601 - matches text : 2020-08-18T14:03:09+09:00 - column name : http_date
        • QS - matches text : GET~~~HTTP/1.1 - column name : request
        • NUMBER - matches text : 200 - column name : response_status
        • BASE16NUM - matches text : 707 - column name : size
      • 패턴 입력 : - []
      • 패턴 최종결과 : %{HOST:host} - %{WORD:user_id} [%{TIMESTAMP_ISO8601:http_date}] %{QS:request} %{NUMBER:response_status} %{BASE16NUM:size}
      • 적용 버튼 클릭
    • 추가
