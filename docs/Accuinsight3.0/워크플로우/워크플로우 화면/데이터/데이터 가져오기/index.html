데이터 가져오기

워크플로우에서 사용될 데이터를 불러옵니다.

노드 메뉴에서 [데이터] 카테고리를 선택하고 우측패널의 [데이터 가져오기]를 선택하여 노드를 추가합니다.

메뉴#

  • 데이터 불러오기 전

    • 이름바꾸기 : 데이터노드의 노드명 변경
    • 즐겨찾기에 추가 : 즐겨찾기에 [데이터 가져오기]노드 추가
    • 데이터 가져오기 : 데이터셋 추가 화면으로 이동
    • 삭제 : 선택된 데이터 노드를 워크플로우에서 제거

  • 데이터 불러오기 후

    • 데이터 보기
      • 전체 화면으로 보기 : 데이터 미리보기 화면으로 이동
      • 팝업으로 보기 : 데이터 미리보기 팝업 호출
    • Wrangler : 선택된 데이터노드에 Wrangler노드를 추가
    • 차트생성 : 차트 생성 화면으로 이동
    • 이름바꾸기 : 데이터노드의 노드명 변경
    • 데이터셋 애셋으로 저장 : 선택된 데이터노드를 데이터셋 애셋에 추가
    • 내보내기 : 데이터셋 내보내기 화면으로 이동
    • 데이터셋 지우기 : 데이터노드에서 데이터를 초기화
    • 즐겨찾기에 추가 : 즐겨찾기에 [데이터 가져오기]노드 추가
    • 삭제 : 선택된 데이터 노드를 워크플로우에서 제거

데이터셋 추가#

  1. 로컬 파일 : 로컬에 있는 data파일을 불러옵니다.
  2. Amazon S3 : AWS S3에 저장된 데이터를 불러옵니다.
  3. RDB : RDBMS에 저장된 데이터를 불러옵니다.
  4. HTTP : HTTP 프로토콜을 사용하여 데이터를 불러옵니다.
  5. Snowflake : SnowFlake에 저장된 데이터를 불러옵니다.
  6. Asset : 다른 워크플로우에서 저장한 데이터셋 애셋을 불러옵니다.
  7. DAapi : DAapi에 저장된 데이터를 불러옵니다.
  • 로컬 파일

  1. 데이터셋 추가 : 파일을 Drag & Drop 하거나 Browes 버튼을 클릭하여 파일을 선택 합니다.
  2. More Actions : 로컬파일 불러오기 설정 팝업 호출
  3. 삭제 : 해당 데이터셋을 삭제
  4. 추가 : 추가대상의 데이터셋을 워크플로우에 로드
  • 로컬파일 불러오기 설정

  1. format : 불러올 파일의 형식 지정(csv, json, xml, text, parquet, orc 중 택1)
  2. Delimiter : 구분자 입력
  3. Option : 사용 가능한 옵션설정(헤더유무, nanValue, nullValue) 설정
    • key: header, value : true/False = 헤더 설정 여부(True : 헤더설정, False : 헤더미설정)
    • key: nullValue, value : 입력값(ex:NULL) = 데이터에 포함된 입력값(NULL)을 Null문자열 처리
    • key: nanValue, value : 입력값(ex:NaN) = 데이터에 포함된 입력값(NaN)을 NaN문자열 처리
  4. Option 추가 : 옵션이 여러개일때 항목을 추가
  5. Option 삭제 : 선택된 옵션 삭제
  6. 취소 : 입력된 내용을 적용하지 않고 설정 팝업을 닫습니다.
  7. 반영하여 추가 : 입력된 내용을 적용 합니다.
  • Amazon S3

  1. 버킷 선택 : S3 버킷 관리에 등록되어 있는 버킷을 선택하여 파일을 확인
  2. S3 버킷 관리 : [S3 버킷 추가 / 관리] 팝업 호출
  3. 추가 : 데이터셋 추가
  4. More Actions : Amazon S3 설정 팝업 호출
  5. 삭제 : 해당 데이터셋을 삭제
  6. 추가 : 추가대상의 데이터셋을 워크플로우에 로드
  • S3 버킷 추가 / 관리

  1. 버킷이름 : 추가할 버킷의 이름
  2. Target Endpoint : 추가할 버킷의 S3 Endpoint
  3. Access Key : 추가할 버킷의 AWS Access Key
  4. Secret Key : 추가할 버킷의 AWS Secret Key
  5. 공개범위 설정 : 추가할 버킷의 공개범위 설정
    • 나만보기 : 로그인한 계정에만 공개
    • 프로젝트 내 공개 : 현재 프로젝트에 공개
    • 글로벌 : 모든 사용자에게 공개
  6. Test Connection : 입력된 정보로 connection을 확인
  7. 저장 : 입력된 정보를 저장
  8. 취소 : 입력된 내용을 초기화
  9. 추가 : 새로운 버킷 목록을 추가
  10. 닫기 : [S3 버킷 추가 / 관리] 팝업을 닫습니다.

  1. DB 선택 : 불러올 dababase를 선택
  2. 검색 : database내의 table검색
  3. 테이블 선택 : 선택된 DB의 테이블을 클릭 (data Preview생성 및 schema 정보를 불러옵니다.)
    • 더블클릭 : SQL Query입력창에 선택된 테이블을 자동입력
  4. SQL Query입력 : 조회 및 추가할 SQL Query 입력
  5. executeQuery : 입력된 SQL문을 실행하여 data Preview생성
  6. 크기변경 : 드래그 하여 SQL Query입력창의 크기 조절
  7. Data Preview : 100건의 데이터를 조회
  8. Schema : 왼쪽 테이블리스트에서 선택된 테이블의 schema정보를 표기
  9. 취소 : SQL 쿼리작성 팝업을 닫습니다.
  10. 적용 : 작성된 SQL문을 적용하여 데이터셋을 추가

  1. URL : 데이터를 불러올 HTTP URL 입력
  2. Return Data : 응답데이터 위치(JsonObject 또는 JsonArray형태에서 사용할 데이터 path. 'resultSet/data', '/'로 구분)

  • HTTP 불러오기 설정

  1. header : HTTP 요청헤더 (항목이 여러개일때 [+]버튼으로 추가)
  2. format : 불러올 파일의 데이터 포맷 선택(json, xml중 택1)
  3. Delimiter : 불러올 데이터의 파싱 구분자 입력
  4. Option : 사용 가능한 옵션설정(헤더유무, nanValue, nullValue) 설정 (항목이 여러개일때 [+]버튼으로 추가)
    • key: header, value : true/False = 헤더 설정 여부(True : 헤더설정, False : 헤더미설정)
    • key: nullValue, value : 입력값(ex:NULL) = 데이터에 포함된 입력값(NULL)을 Null문자열 처리
    • key: nanValue, value : 입력값(ex:NaN) = 데이터에 포함된 입력값(NaN)을 NaN문자열 처리
  5. schema : column 및 dataType 정의 (항목이 여러개일때 [+]버튼으로 추가)
  6. params : 노드에서 사용할 parameter 입력 (항목이 여러개일때 [+]버튼으로 추가)
  7. 취소 : 입력된 내용을 적용하지 않고 설정 팝업을 닫습니다.
  8. 반영하여 추가 : 입력된 내용을 적용 합니다.

    • DescriptiveStatistics
    • Top Values
    • Max Values
    • Min Values
    • Outliers

  13. 필터 : 소팅 (오름차순/내림차순), 필터 (멀티 선택)

    • 취소 : 적용 하지 않고 닫기
    • 적용 : 선택된 필터 적용

  14. 설정 : 데이터셋 추가시 설정한 옵션의 변경

Correlation#

컬럼들 사이의 상관관계

  1. 차트 영역 : 우측패널에 입력된 내용으로 생성된 차트
  2. 타입 선택 : 차트의 타입선택 (Pearson / Spearman 중 선택)
  3. 비교할 컬럼 선택 : 멀티 선택
  4. 차트 생성하기 : 입력된 내용으로 차트 생성

Interactions#

선택된 컬럼으로 차트 생성

  1. 차트 영역 : 우측패널에 입력된 내용으로 생성된 차트
  2. 그래프 타입 : 차트의 타입선택
    • Line
    • Column
    • Bar
    • Pie
    • Secatter
    • Box Plot
  3. 컬럼 선택 (x축) : x축 기준 컬럼
  4. 컬럼 선택 (y축) : y축 기준 컬럼 (선택된 차트에따라 타입, 선택수 제한)
  5. Calculate : 차트 생성 방식 선택
    • Min
    • Max
    • Mean
    • Median
    • Sum
  6. 차트 생성하기 : 입력된 내용으로 차트 생성

데이터셋 애셋으로 저장#

선택된 데이터노드를 데이터셋 애셋으로 저장합니다.

  1. 데이터셋 애셋 이름 : 저장할 데이터셋 애셋 이름
  2. 설명 : 데이터셋 애셋의 설명
  3. 공개범위 설정 : 데이터셋 애셋의 공개범위 설정
    • 나만보기 : 로그인한 계정에만 공개
    • 프로젝트 내 공개 : 현재 프로젝트에 공개
    • 글로벌 : 모든 사용자에게 공개
  4. 적용된 노드 : 선택된 데이터노드를 만들기위해 적용되었던 Acction 노드들
  5. 취소 : 저장하지 않고 팝업 종료
  6. 등록 : 입력된 내용으로 데이터셋 애셋을 생성

내보내기#

선택된 데이터노드를 내보내기

AWS S3#

  1. 버킷 선택 : S3 버킷 관리에 등록되어 있는 버킷을 선택하여 내보내기
  2. S3 버킷 관리 : [S3 버킷 추가 / 관리] 팝업 호출
  3. 폴더 추가 : 선택된 폴더에 새폴더 생성
  4. 추가 : 선택한 폴더 및 파일에 내보내기
  5. 내보내기 경로 : 내보내기할 S3 경로 (자동입력)
  6. 파일명 : 내보내기할 파일명
  7. 파일 확장자 : 내보낼 파일의 형식 지정(csv, json, txt, parquet, orc 중 택1)
  8. 내보내기 규칙 : 같은 이름의 파일이 존재할때 형식 지정
    • 이어 붙임
    • 덮어 씀
    • 저장하지 않고 건너 뜀
    • 에러 처리
  9. 헤더 : 파일의 컬럼명 추가 여부
  10. 우측 패널에 입력된 내용으로 내보내기

  1. JDBC 선택 : JDBC연결 관리에 등록되어 있는 database의 테이블을 조회
  2. JDBC 연결 관리 : [JDBC 연결 관리] 팝업 호출
  3. 선택 : 내보내기할 table 선택
  4. 데이터베이스 / 스키마 : 내보내기할 table의 database명
  5. 테이블명 : 내보내기할 table명
  6. 내보내기 규칙
