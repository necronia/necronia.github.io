Clustering

Clustering은 데이터의 특성에 따라서 자동으로 군집을 만들어 주는 기능입니다.

Clustering 생성#

워크플로우 작업 선택에서 빈 워크플로우를 선택한 후 '다음' 버튼을 클릭하여 워크플로우를 생성합니다.

Clustering 노드 추가#

  1. 워크플로우에서 데이터 노드를 선택합니다.
  2. 노드 메뉴에서 모델링을 선택합니다.
  3. 우측 패널에서 Clustering을 선택합니다.

Clustering 설정#

  1. Clustering 노드를 선택하면 File name, File size, 클러스터 개수를 선택할 수 있는 우측 패널이 나타납니다.
  2. 클러스터 개수를 설정하고 추가 버튼을 클릭하여 Clustering을 실행합니다.

Clustering 진행#

Clustering 작업이 진행중인 화면입니다. 우측 패널의 '중지' 버튼을 클릭하여 작업을 취소할 수 있습니다.

Clustering 완료#

Clustering 이 완료되면 우측 패널에 상태가 성공으로 표시되고 우측 상단에 '결과 보기' 버튼이 보여집니다.
'결과 보기' 버튼을 클릭하거나 clustering 노드를 마우스 우클릭하여 clustering 노드 옵션에서 'Clustering_1 설정하기' 메뉴를 선택하면 Clustering 결과를 조회하고 설정을 업데이트 할 수 있습니다.

Clustering 노드 옵션#

Clustering 노드를 마우스 우클릭하면 clustering 노드 옵션이 나타납니다.

  1. Clustering_1 설정하기
    • Clustering 결과를 조회하고 설정을 업데이트 할 수 있습니다.
  2. 이름 바꾸기
    • 노드 이름을 변경할 수 있습니다.
  3. 즐겨찾기에 추가
    • 우측 패널의 즐겨찾기에 Clustering 노드를 추가합니다.
  4. 즐겨찾기 삭제
    • 우측 패널의 즐겨찾기에서 Clustering 노드를 삭제합니다.
  5. 삭제
    • Clustering 노드를 삭제합니다.

Clustering 결과 보기#

결과 데이터#

클러스터링 결과를 조회할 수 있습니다.

  1. 결과 데이터, 시각화
    • 결과 데이터 : 클러스터별 데이터를 조회할 수 있습니다.
    • 시각화 : 클러스터링 결과 그래프들과 클러스터별 상세 데이터를 조회할 수 있습니다.
  2. 클러스터 목록
    • 클릭하여 클러스터별 결과 데이터를 조회할 수 있습니다.
  3. 업데이트
    • 클러스터 개수를 변경한 후 클릭하면 클러스터링을 업데이트 할 수 있습니다.

모델 노드#

클러스터링이 완료된 후 모델 노드가 생성됩니다.

  1. 모델 노드
    • 클러스터링이 완료된 후 생성된 모델 노드입니다.
  2. 모델 노드 옵션
    • 모델 레지스트리 등록
      • 새 모델 등록
        • AI Runtime에 새 모델을 등록합니다.
        • 등록할 모델이름 입력하고 중복확인 후 확인을 클릭하여 새 모델을 등록합니다.
      • 새 버전으로 등록
        • AI Runtime에 배포된 모델이 있을 경우 해당 모델의 새 버전으로 모델을 등록할 수 있습니다.
        • 배포된 모델 중 하나를 선택한 후 확인을 클릭하여 새 버전으로 모델을 등록합니다.
    • 이름 바꾸기
      • 모델 노드의 이름을 변경합니다.
    • 삭제
      • 모델 노드를 삭제합니다.
  3. 에측용 Workflow 생성
    • 현재 워크플로우 이름에 Predict prefix를 붙여서 예측용 워크플로우를 생성합니다.

예측용 Workflow#

워크플로우 목록에서 예측용 워크플로우 선택하여 불러옵니다.

  1. 데이터 가져오기를 클릭하여 가져올 데이터를 선택합니다.

  2. 데이터 가져오기가 완료된 후 모델 노드를 선택하고 우측 패널에서 예측 버튼을 클릭합니다.

  3. 예측 완료 후 예측 결과 데이터 노드가 생성됩니다.
