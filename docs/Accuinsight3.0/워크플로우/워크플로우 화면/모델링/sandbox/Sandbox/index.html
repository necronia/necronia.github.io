Sandbox

Sandbox 선택#

Sandbox는 Rstudio 또는 Jupyter Lab과 같은 분석 툴이 설치된 가상 컴퓨팅 환경입니다.

[워크플로우 작업 선택]에서 [Sandbox 바로가기]를 통해 분석 환경 설정으로 이동하여 생성을 할 수 있습니다.

Sandbox 분석 환경 설정#

Sandbox 생성에 필요한 설정들을 입력 후 Sandbox를 생성합니다.

  1. 분석 모델 : Rstudio 또는 Jupyter lab(3.7) 중 선택합니다.
  2. 자원 설정 : Sandbox가 사용할 자원을 Small, Medium, Large, 직접 입력 중 선택합니다.
    • 직접 입력을 선택하는 경우 Dropdown 메뉴를 통해서 CPU, Memory 설정을 할 수 있습니다.
    • 해당 프로젝트에서 GPU 자원 사용이 가능하도록 설정 된 경우 GPU 설정을 추가할 수 있습니다.
    • GPU 설정 추가 선택 후 Dropdown 메뉴를 통해서 GPU Core를 설정 할 수 있습니다.
  3. Rstudio / Jupyter 비밀번호 설정
    • Rstudio / Jupyter Lab 접속 시 사용을 할 비밀 번호를 입력합니다.
    • 비밀 번호는 영문, 특수문자 조합 8자 이상이어야 합니다.

Sandbox 관리#

워크플로우의 생성된 Sandbox 노드를 확인 할 수 있습니다.

  1. 생성된 Sandbox 노드 선택 시 우측패널에 Sandbox와 관련된 메뉴가 활성화 되며, 노드를 더블 클릭 시 상세 화면이 표시 됩니다.
  2. Sandbox 파일시스템에서 데이터 파일을 워크플로우 데이터 노드로 가져 올 수 있도록 데이터 추출을 위한 파일 목록이 표시 됩니다.
    • 추출 가능한 데이터 파일 포맷 : csv, json, xml, txct, log, parquet, orc
  3. Sandbox 에서 생성된 모델을 워크플로우 모델 노드로 가져올 수 있습니다.
  4. 브라우저의 새창으로 Jupyter Lab 페이지로 이동합니다.
  5. Sandbox 의 상태 확인 및 변경을 할 수 있습니다.
    • Pending : Sandbox 환경 생성 대기 중
    • Waiting : Sandbox 환경 생성 중
    • Running : Sandbox 환경에 접속 가능한 정상 실행중인 상태입니다.
    • Stop : Sandbox 환경이 중지 상태이며, 접속 불가능.

Sandbox 상세 정보#

Sandbox 상세페이지는 해당 컨테이너의 상세 정보를 조회하고 분석툴에서 생성한 파일을 조회하고 저장소로 저장할 수 있습니다.

1. Sandbox 정보#

- 워크스페이스 생성시 설정된 정보와 현재 컨테이너 상태, 발생한 이벤트 정보를 조회합니다.- Endpoint URL 바로가기 (분석툴 새창 open)지원, Jupyter/R 환경 생성 시 설정한 비밀번호 숨김/확인 기능을 제공합니다.  1) 분석 환경에 대한 정보 확인    - 접속 정보 : Rstudio 또는 Jupyter lab 접속 패스워드            - 컨테이너 이미지 : Sandbox 컨테이너 이미지의 저장 경로 정보     2) Sandbox의 CPU, GPU, MEMORY 자원 정보를 확인 및 수정 
> 자원 설정 : stop 상태일 때만 수정 가능    3) 컨테이너의 상태 히스토리, 발생 이벤트 확인

3) 소스를 새로운 이름 또는 기존 저장된 소스에 새로운 버전을 발행하여 저장되며, 버전은 시스템에서 자동으로 발행됩니다.
4) 저장하는 소스의 공유범위 설정     - 나만 보기 : 소스 저장소를 생성한 사용자만 볼 수 있습니다.    - 프로젝트 내 공개 : 소스 저장소를 생성한 사용자가 포함된 프로젝트의 구성원에게 공개    - 전체 공개 : 전체 구성원에게 모두 공개

Experiment > AccuInsight SDK#

Experiment에 학습된 모델을 등록하기 위한 Python sdk인 Accuinsight 를 제공합니다.
Accuinsight 는 학습에 사용된 machine learning platform에 따라 3가지 api로 구분되며, 사용자 편의를 위한 utils 를 포함합니다. 

-  ___ML___ API - Scikit-learn / Xgboost / lightGBM   - ___keras___ API - Keras   - ___tensorflow___ API - Tensorflow 2.0 (keras)  - ___utils___

Tensorflow 2.0의 estimator로 작성된 모델과 Pytorch는 현재 지원하지 않습니다.

 with accu.add_experiment(model_name, X_test, y_test) as exp:   exp.log_params('alpha')   exp.log_params('beta')   exp.log_metrics('MSE', mse)     exp.log_metrics('MAE', mae)   exp.log_tag('model-test') ``` - __add_experiment__ 는 학습된 모델을 등록하는 데 사용되는 class로, _with_ 구문에서 아래의 메소드를 수행합니다.     - (_method_) __log_params(__ '_hyperparameter name_' __)__       모델 학습에 사용된 모델의 hyperparameter를 기록합니다.     - (_method_) __log_metrics(__ _'metric name'_ , _defined_metric_ __)__       모델로 예측을 수행한 후, 계산된 metric(_defined_metric_)을 기록하고자 하는 이름('_metric name_')과 함께 입력해줍니다.     - (_method_) __log_tag(__ '_model description_' __)__       모델에 대한 간단한 설명을 기록합니다.   - ___Arguments___   _model_name_: 학습된 모델의 이름 _X_test_: 평가에 사용될 input dataset의 이름 _y_test_: 평가에 사용될 target dataset의 이름  <u>Scikit-learn의  SVM으로 모델링을 진행할 경우, 다음을 유의하시기 바랍니다.</u>     - multiclass classification에서 __SVC__ 를 사용하여 모델링 할 경우, `probability = True ` 옵션 설정이 필요합니다.    - multiclass classification에서 __LinearSVC__ 를 사용하여 모델링 한 경우, `predict_proba()` 를 제공하지 않아 ROC/Precision-recall curve가 제공되지 않습니다.         3. __keras__ API    [[link] keras API example](https://github.com/accuinsight)       (_class_)  __accuinsight()__ ```python from Accuinsight.Lifecycle.keras import accuinsight
 accu = accuinsight() ``` - accuinsight() class를 임의의 변수(_accu_)로 선언합니다.  (_class_) __autolog(__ _tag_ = '_model description_' __)__ ```python accu.autolog(tag = 'keras-test') ``` - __autolog__ 는 keras의 `fit()` 메소드를 수행하기 이전에 선언한 후,  `fit()` 을 통해 모델을 학습시키면, 학습 과정에서 자동으로 Experiment를 등록합니다.         - __autolog__ 를 선언하면, 모델 학습 종료 시점에 가장 좋은 metric을 갖는 epoch의 weight가 자동으로 설정됩니다. 단, 이 때 `fit()` 메소드에 입력된 epoch만큼 학습이 진행되므로 빠른 학습 종료를 위해서는  _early_stopping_ 콜백을 추가해야 합니다.     - __autolog__ 는 가장 좋은 metric을 기록한 epoch에서의 모델 정보와 weight를 저장합니다. 따라서 이전에 저장된 모델을 불러와 공동 작업자들과 공유하거나, 재학습을 수행할 수 있습니다. 모델을 불러오는 메소드는 ___utils___ 항목을 참고하세요.      - ___Arguments___     _tag_: 모델에 대한 간단한 설명           > `model.compile()` 시, _metrics_ 리스트에서 처음으로 입력 된 평가지표를 활용하여 모델을 훈련합니다. 따라서 모델 훈련 시 기준이 되는 평가 지표를 먼저 입력해야 합니다.  예를 들어 위와 같이 모델을 컴파일 한 경우, 모델 훈련 시 _f1_score_ 를 참조하여 가장 좋은 _f1_score_ 를 기록한 모델을 기록하고, 적합시킵니다.            ```python     model.compile(metrics = [f1_score, accuracy])     ```
