JupyterLab 확장

Git#

Git을 사용한 버전 관리

  • 참조 - https://github.com/jupyterlab/jupyterlab-git

1. 주요 내용#

  • disableBranchWithChanges : 커밋되지 않은 변경 내용을 덮어 쓰거나 손실하지 않도록 보호합니다.
  • historyCount : 가장 최근부터 시작하여 기록 로그에 표시되는 커밋 수를 표시합니다.
  • refreshInterval : 변경 사항에 대해 파일 시스템을 폴링하는 간격(밀리초)을 의미합니다.
  • simpleStaging : 간단한 스테이징 개념을 가능하게하며, 변경 사항이 시각적으로 간단하게 표현됩니다.

1) Git clone 방법

  • gitlab, github 등 기타 repository 관련 프로그램 clone
  • 자체 branch 및 repository 관리 기능을 제공합니다.(push 불가능)

Debugger#

Jupyter 소스 파일을 위한 시각적 debugger
참조 - https://github.com/jupyterlab/debugger

1. 주요 내용#

  • python 구문 사용 시, 단계별 디버그 확인
  • Kernel은 xpython만 가능
  • xpython은 여러개의 라이브러리가 포함된 python 인터프리터 (참고 - http://xpython.org/)

1) Kernel 변경 방법

  • 최초 진입 시, Launcher에서 xpython 선택
  • python3로 작업 중, Kernel 변경 가능

LSP#

IDE의 다양한 유사 기능을 제공합니다.

  • 코드 탐색
  • 호버 제안
  • 린터
  • 진단
  • 커널 없는 자동 완성 등
  • 참조 - https://github.com/krassowski/jupyterlab-lsp

1. 주요 내용#

  • Language server 연동으로 폐쇄망에서는 사용이 불가합니다.

  • 하단에 [Fully initialized] 문구가 표시 되어야 정상 사용이 가능합니다.

2. 상세 기능 소개#

  • Highlight References - 변수, 함수 등에 커서를 놓으면 모든 사용법이 강조 표시됩니다.

  • Hover - 코드 조각 위로 마우스 오버시 밑줄이 나타나면 Ctrl 선택 기능 / 클래스 서명, 모듈 설명서 또는 언어 서버가 제공하는 기타 정보가 포함 된 툴팁을 얻을 수 있습니다.

  • Diagnostics - 심각한 오류에는 빨간색 밑줄이 있고 경고는 주황색으로 표기하며, 밑줄이 표시된 코드 위로 마우스를 가져 가면 자세한 메시지가 표시됩니다.

  • Jump to Definition - 상황에 맞는 메뉴 항목을 사용하여 정의로 이동합니다.

  • Automatic Completion - 특정 문자 (예 : '.' 파이썬에서 (도트), 자동 완성을 트리거합니다.

  • Automatic Signature Suggestions - 기능 서명이 자동으로 표시됩니다.

  • Kernel-less Autocompletion - 커널 실행 없이 고급 정적 분석 자동 완성 기능을 제공합니다.

  • Rename - 노트북과 파일 편집기에서 변수, 함수 등의 이름을 재설정하며, 상황에 맞는 메뉴 옵션 또는 [F2] 바로가기를 사용하여 호출이 가능합니다.

  • Diagnostics panel - 진단 패널을 사용하여 진단항목 정렬 후 이동하고, JupyterLab 명령 팔레트 또는 컨텍스트 메뉴에서 "진단 패널 표시"를 검색합니다.
